1. Der SID — Seele des C64-Klangs

Der MOS Technology 6581 SID (Sound Interface Device) ist unbestreitbar einer der einflussreichsten Soundchips der Computergeschichte. Entworfen von Bob Yannes, der später Ensoniq gründete und den legendären Synthesizer-Sound der späten 1980er-Jahre mitprägte, ist der SID ein vollständiger analoger Synthesizer auf einem einzigen Chip. Diese Tatsache erklärt, warum C64-Musik bis heute einen unverwechselbaren, lebendigen Klangcharakter hat, der digitale Soundchips nicht replizieren können.

Der SID bietet drei vollständig unabhängige Stimmen (Voices). Jede Stimme verfügt über einen eigenen Oszillator mit vier Wellenformen, einen ADSR-Hüllkurven-Generator und eine Gate-Funktion. Dazu kommt ein gemeinsamer analoger Filter mit einstellbarer Cutoff-Frequenz, Resonanz und drei wählbaren Charakteristiken (Low-Pass, Band-Pass, High-Pass). Die analogen Komponenten des SID — insbesondere der Filter — erzeugen genau die "Unschärfen" und "Wärme", die den C64-Sound so charakteristisch machen.

Zwei Chip-Revisionen existieren: Der 6581 (1982–1985) hat durch seine niedrigere Versorgungsspannung einen stärkeren, raueren Filter-Charakter und erzeugt leises Rauschen im Ausgangssignal. Der 8580 (1987–1994) ist elektrisch "sauberer" mit niedrigerem Rauschboden, aber weniger markanter Filtercharakteristik. Viele Musiker und Puristen bevorzugen den 6581.

2. SID-Register-Map ($D400–$D41C)

Der SID belegt 29 Register im Adressbereich $D400 bis $D41C. Die Registers für die drei Stimmen sind je 7 Bytes groß und aufeinanderfolgende Blöcke:

2.2 Control-Register Bits (Offset +4)

BitNameFunktion
0GATE1 = Note starten (Attack→Decay→Sustain); 0 = Note loslassen (Release-Phase)
1SYNCSynchronisiert Oszillator mit Vorgänger-Oszillator (1→3, 2→1, 3→2)
2RINGRing-Modulation mit Vorgänger-Oszillator (ersetzt Triangle-Wellenform)

⚠ Mehrere Wellenformen gleichzeitig

Mehrere Wellenform-Bits können gleichzeitig gesetzt werden. Das Ergebnis ist nicht eine Addition, sondern eine bitweise AND-Verknüpfung der Wellenform-Ausgaben. Dies erzeugt härtere, eigenartigere Klänge — von Demo-Codern für spezielle Effekte genutzt. Combination TRI+SAW klingt nach einem dumpfen, metallischen Ton.

3. Frequenzberechnung und Noten-Tabelle

3.1 Die Frequenzformel

Der SID-Frequenzwert wird aus der gewünschten Ausgangsfrequenz mit folgender Formel berechnet:

Frequenzregister = (Fout × 16777216) / Fclock

PAL-Systemtakt:  Fclock = 985.248 Hz
NTSC-Systemtakt: Fclock = 1.022.727 Hz

Beispiele (PAL):
  C4 (261,63 Hz): Reg = 261,63 × 16777216 / 985248 = 4455 = $1167
  A4 (440,00 Hz): Reg = 440,00 × 16777216 / 985248 = 7493 = $1D45
  C5 (523,25 Hz): Reg = 523,25 × 16777216 / 985248 = 8910 = $22CE

4. ADSR — Die Klang-Hüllkurve

Die ADSR-Hüllkurve (Attack, Decay, Sustain, Release) bestimmt, wie sich die Lautstärke einer Note über die Zeit verändert. Sie ist das wichtigste Werkzeug zur Gestaltung des Klangcharakters eines Instruments.

Lautstärke (0-15)
    15 ──────┐ Decay ─ Sustain ────────────────
             │       /                         \
         A  ╱        │                          \ Release
            │                                    \
     0 ─────┴────────┴────────────────────────────┴──→ Zeit
         Attack   Note gehalten                Note losgelassen

5. Vollständige Sound-Routinen

5.1 Einfacher Ton auf Voice 1

; =====================================================
; Ton spielen: Note C4 (261 Hz) mit Sawtooth-Welle
; ADSR: Piano-ähnlich (kurzer Attack, langer Decay)
; =====================================================

PlayNote_C4:
        ; Master-Lautstärke auf 15 (falls noch nicht gesetzt)
        lda #$0f
        sta $d418

        ; ADSR: Attack=0, Decay=9, Sustain=0, Release=9
        lda #$09        ; Attack=0 (Hi-Nibble), Decay=9 (Lo-Nibble)
        sta $d405
        lda #$09        ; Sustain=0, Release=9
        sta $d406

        ; Frequenz: C4 ≈ $1167 (PAL)
        lda #$67        ; Frequenz Low
        sta $d400
        lda #$11        ; Frequenz High
        sta $d401

        ; Sawtooth-Welle + Gate AN
        lda #%00100001  ; SAW (Bit 5) + GATE (Bit 0)
        sta $d404
        rts

; Note beenden (Gate aus → Release-Phase):
StopNote_V1:
        lda $d404
        and #%11111110  ; Gate-Bit löschen
        sta $d404
        rts

5.2 Percussion-Sounds

; =====================================================
; Schlagzeug-Sounds auf Voice 3
; Bassdrum: Rauschen + fallende Frequenz
; =====================================================

PlayBassDrum:
        ; Voice 3: Noise, kurze ADSR
        lda #$00        ; Attack=0, Decay=0
        sta $d413
        lda #$00        ; Sustain=0, Release=0
        sta $d414

        ; Hohe Anfangsfrequenz für "Knall"-Charakter
        lda #$ff
        sta $d40e       ; Freq Lo
        lda #$08
        sta $d40f       ; Freq Hi

        ; Triangle + Gate (Triangle klingt für Bassdrum warm)
        lda #%00010001  ; TRI + GATE
        sta $d412

        ; Frequenz wird per Hauptschleife auf 0 heruntergezogen
        rts

PlaySnare:
        ; Noise mit mittlerer Frequenz und kurzem ADSR
        lda #$04        ; Attack=0, Decay=4
        sta $d413
        lda #$02        ; Sustain=0, Release=2
        sta $d414
        lda #$80        ; Mittlere Frequenz
        sta $d40e
        lda #$1a
        sta $d40f
        lda #%10000001  ; NOI + GATE
        sta $d412
        rts

PlayHiHat:
        lda #$02        ; Attack=0, Decay=2
        sta $d413
        lda #$01        ; Sustain=0, Release=1
        sta $d414
        lda #$ff
        sta $d40e       ; Hohe Frequenz = helles Rauschen
        lda #$7f
        sta $d40f
        lda #%10000001  ; NOI + GATE
        sta $d412
        rts

6. Voice 3 als LFO — Vibrato-Effekt

Eine der mächtigsten SID-Techniken ist die Nutzung von Voice 3 als Low-Frequency Oscillator (LFO). Indem Voice 3 auf eine sehr niedrige Frequenz eingestellt und über Register $D41B ausgelesen wird, können Voice 1 und 2 mit einem Vibrato- oder Tremolo-Effekt moduliert werden.

; =====================================================
; Voice 3 als LFO für Vibrato auf Voice 1
; Voice 3: sehr langsam (~5 Hz), Triangle, kein Gate
; =====================================================

InitLFO:
        ; Voice 3: LFO-Frequenz (~5 Hz PAL)
        ; Reg = 5 × 16777216 / 985248 ≈ 85 = $55
        lda #$55        ; LFO-Frequenz Lo
        sta $d40e
        lda #$00        ; Hi = 0
        sta $d40f
        ; Control: Triangle OHNE Gate (kein Ton, nur Wellenform)
        lda #%00010000  ; TRI, kein GATE
        sta $d412
        ; Voice 3 stumm schalten (nicht im Audio-Mix)
        lda $d418
        ora #%10000000  ; Bit 7 setzen
        sta $d418
        rts

; Im Haupt-IRQ (50 Hz): Vibrato auf Voice 1 anwenden
ApplyVibrato:
        ; LFO-Wert lesen (0-255, repräsentiert Sinuswelle)
        lda $d41b       ; Voice-3-Waveform-Ausgang

        ; Auf ±16 Frequenzschritte skalieren
        lsr             ; / 2
        lsr             ; / 4  (0-63)
        lsr             ; / 8  (0-31)
        sec
        sbc #16         ; -16 bis +15 (signed)

        ; Zu Basis-Frequenz addieren (16-Bit-Addition)
        clc
        adc BaseFreqLo
        sta $d400       ; Voice 1 Freq Lo

        lda BaseFreqHi
        adc #0          ; Carry übertragen
        sta $d401       ; Voice 1 Freq Hi
        rts

BaseFreqLo: !byte $64   ; C4 Lo
BaseFreqHi: !byte $11   ; C4 Hi

7. Einfacher Musik-Player

; =====================================================
; Einfacher 1-stimmiger Musik-Player
; Format: Note-Index (0=Pause, 1=C3...), Dauer (Frames)
; Aufruf: 50x/Sekunde im Raster-IRQ
; =====================================================

; Player-Variablen (Zero Page)
mus_note_dur  = $70   ; Verbleibende Dauer der aktuellen Note
mus_pos       = $71   ; Position im Melodie-Array (Low-Byte)
mus_pos_hi    = $72   ; Position (High-Byte)

; Player initialisieren
MusicInit:
        lda #Melody
        sta mus_pos_hi
        lda #1
        sta mus_note_dur ; Sofort erste Note laden
        ; Lautstärke setzen
        lda #$0f
        sta $d418
        rts

; Player-Update (einmal pro Frame aufrufen)
MusicUpdate:
        dec mus_note_dur
        bne @playing    ; Noch nicht fertig

        ; Nächste Note laden
        ldy #0
        lda (mus_pos),y ; Note-Index
        bmi @end_of_music   ; $FF = Ende

        beq @play_rest  ; 0 = Pause

        ; Frequenz aus Tabelle laden
        sec
        sbc #1          ; Index 1 = C3 = Tabellen-Offset 0
        tax
        lda NoteTabLo,x
        sta $d400
        lda NoteTabHi,x
        sta $d401

        ; Sawtooth + Gate
        lda #$21        ; SAW + GATE
        sta $d404
        bne @load_dur

@play_rest:
        ; Gate aus für Pause
        lda $d404
        and #$fe
        sta $d404

@load_dur:
        ; Dauer laden (Byte 2 der Note)
        iny
        lda (mus_pos),y
        sta mus_note_dur

        ; Pointer weitersetzen (+2)
        lda mus_pos
        clc
        adc #2
        sta mus_pos
        bcc @playing
        inc mus_pos_hi

@playing:
        rts

@end_of_music:
        ; Lied wiederholen
        lda #Melody
        sta mus_pos_hi
        lda #1
        sta mus_note_dur
        rts

; Melodie-Daten: Note-Index, Dauer (in Frames @ 50 Hz)
; Note 1=C3, 2=C#3, ..., 12=B3, 13=C4, ...
Melody:
        !byte 13,12  ; C4, 12 Frames (Viertel bei ~100 BPM)
        !byte 15,12  ; D4
        !byte 17,12  ; E4
        !byte 13,24  ; C4, halbe Note
        !byte  0,12  ; Pause
        !byte 17,12  ; E4
        !byte 18,12  ; F4
        !byte 20,24  ; G4
        !byte $ff    ; Ende-Marker
