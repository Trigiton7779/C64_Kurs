Bitmap-Grafik

Fortgeschritten 🕑 ca. 8 Stunden Voraussetzungen: Stufen 0–4

🎯 Lernziele dieser Stufe

  • Den Hires-Bitmap-Modus (320×200) vollständig einrichten und nutzen
  • Den Multicolor-Bitmap-Modus (160×200, 4 Farben pro Zelle) beherrschen
  • Pixel-genaue Zeichenoperationen programmieren
  • Das FLI-Konzept (Flexible Line Interpretation) für mehr Farben verstehen
  • Grafiken im Koala-Format laden und anzeigen
  • Optimierte Bitmap-Routinen für Performance schreiben

1. Bitmap-Grafik — Pixel für Pixel

Während der Zeichenmodus auf einem vordefinierten Raster aus 8×8-Pixel-Blöcken basiert, erlauben die Bitmap-Modi des C64 pixel-genaue Kontrolle über das gesamte Bild. Diese Fähigkeit ist Grundlage für alle hochauflösenden Grafiken, Ladebilder und viele fortgeschrittene Demo-Effekte.

Der C64 bietet zwei Bitmap-Modi: Hires (320×200, 2 Farben pro 8×8-Zelle) und Multicolor (160×200, 4 Farben pro 8×8-Zelle). Beide teilen dieselbe Grundidee: Die Bitmap enthält rohe Pixel-Bits, während Screen-RAM und Color-RAM die Farbinformationen liefern.

1.1 Speicherbedarf im Überblick

ModusBitmapScreen-RAMColor-RAMGesamt
Hires Bitmap8.000 Bytes1.000 Bytes—9.000 Bytes
Multicolor Bitmap8.000 Bytes1.000 Bytes1.000 Bytes10.000 Bytes
FLI (Hires)8.000 Bytes8.000 Bytes1.000 Bytes17.000 Bytes

2. Hires-Bitmap-Modus (Standard Bitmap)

2.1 Aktivierung

Der Hires-Bitmap-Modus wird durch Setzen von Bit 5 (BMM) in Register $D011 aktiviert:

; Hires-Bitmap-Modus aktivieren:
lda $d011
ora #%00100000  ; Bit 5 (BMM) setzen
sta $d011

; MCM (Multicolor) muss 0 sein:
lda $d016
and #%11101111  ; Bit 4 löschen
sta $d016

2.2 Speicher-Layout

Register $D018 Bit 3 bestimmt, wo die Bitmap-Daten innerhalb der VIC-Bank liegen:

  • Bit 3 = 0: Bitmap bei Offset $0000 der VIC-Bank
  • Bit 3 = 1: Bitmap bei Offset $2000 der VIC-Bank

Bei Standard VIC-Bank 0: Bitmap entweder bei $0000 oder $2000.

Screen-RAM bestimmt die Farben der einzelnen 8×8-Zellen:

  • High-Nibble (Bits 4–7): Farbe der gesetzten Bits (Pixel "an")
  • Low-Nibble (Bits 0–3): Farbe der gelöschten Bits (Pixel "aus" = Hintergrund)
📝 Hinweis: 2 Farben pro Zelle

Im Hires-Bitmap-Modus können nur 2 Farben pro 8×8-Pixel-Zelle verwendet werden — nicht pro Pixel. Das bedeutet: Innerhalb einer Zelle sind alle "1"-Pixel gleiche Farbe, alle "0"-Pixel ebenfalls. Für mehr Farben pro Zelle wird FLI benötigt.

2.3 Bitmap-Byte-Layout

Die Bitmap belegt 8000 Bytes: 40 Zellen horizontal × 25 Zellen vertikal × 8 Bytes pro Zelle. Die Adressierung folgt einem verschachtelten Schema:

Byte-Adresse eines Pixels an (px, py):
  Zellen-Spalte  = px / 8
  Zellen-Zeile   = py / 8
  Pixel-in-Zelle-X = px MOD 8
  Pixel-in-Zelle-Y = py MOD 8

  Byte-Offset = (Zellen-Zeile × 320) + (Zellen-Spalte × 8) + Pixel-in-Zelle-Y
  Bit-Position = 7 - Pixel-in-Zelle-X

Beispiel: Pixel (100, 75)
  Zellen-Spalte  = 100/8 = 12
  Zellen-Zeile   = 75/8  = 9
  PinZ-X = 100 MOD 8 = 4
  PinZ-Y = 75 MOD 8  = 3

  Byte-Offset = (9×320) + (12×8) + 3 = 2880 + 96 + 3 = 2979
  Bit = 7 - 4 = 3 → Maske = %00001000 = $08

2.4 Vollständige Hires-Bitmap-Initialisierung

; =====================================================
; Hires-Bitmap initialisieren
; Bitmap bei $2000, Screen-RAM bei $0400, VIC-Bank 0
; $D018 = $18: Screen-Offset 1 ($0400), Bitmap-Offset 4 ($2000)
; =====================================================

InitHiresBitmap:
        ; 1. Bitmap-RAM löschen (8000 Bytes bei $2000)
        lda #0
        ldx #0
@clr:   sta $2000,x     ; Seite $20
        sta $2100,x
        sta $2200,x
        sta $2300,x
        sta $2400,x
        sta $2500,x
        sta $2600,x
        sta $2700,x
        sta $2800,x
        sta $2900,x
        sta $2a00,x
        sta $2b00,x
        sta $2c00,x
        sta $2d00,x
        sta $2e00,x
        sta $2f00,x
        ; Verbleibende 1984 Bytes (16*256=4096, 8000-4096=3904... besser per Schleife)
        inx
        bne @clr

        ; 2. Screen-RAM: Farben initialisieren (weiß auf schwarz: $10)
        ;    High-Nibble = 1 (weiß für Pixel=1), Low-Nibble = 0 (schwarz für Pixel=0)
        lda #$10
        ldx #0
@scr:   sta $0400,x
        sta $0500,x
        sta $0600,x
        inx
        bne @scr
        ldx #0
@scr2:  sta $06e8,x
        inx
        cpx #232
        bne @scr2

        ; 3. VIC-II auf Bitmap-Modus schalten
        ; $D018 = $18: Bits 7-4 = 0001 (Screen bei $0400), Bit 3 = 1 (Bitmap bei $2000)
        lda #$18
        sta $d018

        ; $D011: BMM (Bit5) setzen, 25 Zeilen, Y-Scroll=3
        lda #%00111011  ; $3B
        sta $d011

        ; $D016: kein MCM, 40 Spalten, X-Scroll=0
        lda #%00001000  ; $08
        sta $d016
        rts

2.5 Pixel setzen und löschen

; Pixel setzen (1) oder löschen (0)
; Eingabe: $60 = px (0-319), $62 = py (0-199), $64 = Aktion (0=löschen, 1=setzen)

SetPixel:
        ; Byte-Adresse berechnen
        ; Zellen-Zeile = py/8, Zellen-Spalte = px/8
        lda $62         ; py
        lsr
        lsr
        lsr             ; py/8 = Zellen-Zeile (0-24)
        tax             ; X = Zellen-Zeile
        lda RowOffsLo,x ; Low-Byte des Zeilen-Offsets
        sta $fb
        lda RowOffsHi,x ; High-Byte
        sta $fc         ; $fb/$fc = Zellen-Zeile * 320

        ; Zellen-Spalte * 8 addieren
        lda $60         ; px
        and #%11111000  ; px & ~7 = (px/8)*8
        clc
        adc $fb
        sta $fb
        bcc @noc1
        inc $fc
@noc1:
        ; py MOD 8 addieren
        lda $62
        and #$07        ; py & 7 = py MOD 8
        clc
        adc $fb
        sta $fb
        bcc @noc2
        inc $fc
@noc2:
        ; Bitmap-Basis $2000 addieren
        lda $fc
        clc
        adc #$20        ; Hi-Byte von $2000
        sta $fc

        ; Bit-Maske berechnen: Bit = 7 - (px MOD 8)
        lda $60
        and #$07        ; px MOD 8
        tax
        lda BitMasks,x  ; Bit-Maske

        ; Pixel setzen oder löschen
        ldy #0
        bit $64
        beq @clear_pixel
        ora ($fb),y     ; Bit setzen (OR)
        bne @store
@clear_pixel:
        eor #$ff        ; Invertieren für AND-Maske
        and ($fb),y     ; Bit löschen
@store: sta ($fb),y
        rts

BitMasks: !byte $80,$40,$20,$10,$08,$04,$02,$01

; Zeilen-Offset-Tabelle: Zellen-Zeile N * 320 (Lo/Hi)
RowOffsLo:
        !byte <(0*320),<(1*320),<(2*320),<(3*320),<(4*320)
        !byte <(5*320),<(6*320),<(7*320),<(8*320),<(9*320)
        !byte <(10*320),<(11*320),<(12*320),<(13*320),<(14*320)
        !byte <(15*320),<(16*320),<(17*320),<(18*320),<(19*320)
        !byte <(20*320),<(21*320),<(22*320),<(23*320),<(24*320)
RowOffsHi:
        !byte >(0*320),>(1*320),>(2*320),>(3*320),>(4*320)
        !byte >(5*320),>(6*320),>(7*320),>(8*320),>(9*320)
        !byte >(10*320),>(11*320),>(12*320),>(13*320),>(14*320)
        !byte >(15*320),>(16*320),>(17*320),>(18*320),>(19*320)
        !byte >(20*320),>(21*320),>(22*320),>(23*320),>(24*320)

3. Multicolor-Bitmap-Modus

3.1 Aktivierung

; Multicolor-Bitmap aktivieren:
lda #%00111011  ; $D011: BMM setzen
sta $d011
lda #%00011000  ; $D016: MCM (Bit4) + 40 Spalten
sta $d016

3.2 Vier Farben pro Zelle

Im Multicolor-Bitmap-Modus ist jedes Pixel 2 Bits breit. Pro 8×8-Pixel-Zelle (tatsächlich 4×8 effektive Pixel) gibt es 4 mögliche Werte:

Bit-PaarFarbquellePro Zelle individuell?
%00$D021 (globale Hintergrundfarbe)Nein
%01Screen-RAM High-NibbleJa
%10Screen-RAM Low-NibbleJa
%11Color-RAM (4 Bit, $D800+Offset)Ja

In jedem Byte der Bitmap kodieren je 2 aufeinanderfolgende Bits einen Pixel: Bits 7–6 = Pixel 1 (ganz links), Bits 5–4 = Pixel 2, Bits 3–2 = Pixel 3, Bits 1–0 = Pixel 4 (ganz rechts). Pro Byte also 4 Pixel.

3.3 Koala-Format

Das Koala Painter-Format ist das verbreitetste C64-Bitmapformat und enthält alle Daten für einen Multicolor-Bitmap-Screen in einer Datei:

OffsetGrößeInhalt
$00002 BytesLadeadresse ($6000 für Standard-Koala)
$00028.000 BytesBitmap-Daten
$1F421.000 BytesScreen-RAM-Daten
$232A1.000 BytesColor-RAM-Daten
$27121 ByteHintergrundfarbe ($D021)
 
; Koala-Bild anzeigen (Annahme: bereits bei Standard-Adressen geladen)
; Bitmap bei $6000, Screen bei $7F40, Color bei $8328, BG bei $8710

ShowKoala:
        ; Multicolor-Modus aktivieren
        lda #$3b : sta $d011    ; BMM + 25 Zeilen + Y-Scroll=3
        lda #$18 : sta $d016    ; MCM + 40 Spalten

        ; Bitmap bei $6000: VIC-Bank auf $4000-$7FFF (Bank 1)
        lda $dd00
        and #%11111100
        ora #%00000010  ; Bank 1 ($4000-$7FFF)
        sta $dd00

        ; Screen bei $7F40: Offset in Bank 1 = $7F40-$4000 = $3F40
        ; $D018 Bits 7-4: $3F40 / $0400 = 15 → Wert $F → Bits 7-4 = %1111
        ; Bitmap bei $6000: Offset $6000-$4000 = $2000 → Bit 3 = 1
        lda #%11111000  ; Screen=$3C00 in Bank... (vereinfachte Version)
        ; Korrekt: Screen $7F40 - VIC-Bank-Basis $4000 = $3F40
        ; $3F40 / $0400 = 15 (gerundet auf nächste $0400-Grenze = $3C00)
        ; Hinweis: Koala lädt normalerweise auf feste Adressen, Details je nach Version

        ; Hintergrundfarbe setzen
        lda $8710
        sta $d021

        ; Color-RAM füllen (1000 Bytes von $8328 nach $D800 kopieren)
        ldx #0
@clr1:  lda $8328,x : sta $d800,x
        lda $8428,x : sta $d900,x
        lda $8528,x : sta $da00,x
        inx : bne @clr1
        ldx #0
@clr2:  lda $8628,x : sta $dae8,x
        inx : cpx #232 : bne @clr2

        rts

4. Linien und geometrische Primitive

4.1 Bresenham-Linie in Assembler

Der Bresenham-Linienalgorithmus ist die Standardmethode zum effizienten Zeichnen von Linien auf Rasterdisplays. Er kommt ohne Fließkomma-Arithmetik aus, was ihn ideal für 6510-Assembler macht:

; Bresenham-Linie zeichnen
; Eingabe: $60/$61 = x0/y0, $62/$63 = x1/y1
; Setzt Pixel in Hires-Bitmap bei $2000

DrawLine:
        ; Differenzen berechnen
        lda $62 : sec : sbc $60 : sta dx  ; dx = x1 - x0 (vorzeichenbehaftet)
        lda $63 : sec : sbc $61 : sta dy  ; dy = y1 - y0

        ; Steigung bestimmen und Hauptachse wählen
        ; (vereinfachte Version für positive dx, dy)

        lda dx : sta step_count  ; Schrittzahl = max(dx, dy)
        lda dy : cmp dx
        bcc @dx_major
        sta step_count

@dx_major:
        ; Schritt-Schleife (für horizontale Hauptachse)
        lda #0 : sta err
        lda $60 : sta cx    ; cx = x0
        lda $61 : sta cy    ; cy = y0
        ldy step_count

@loop:
        ; Pixel an (cx, cy) setzen
        lda cx : sta $60
        lda cy : sta $62
        lda #1 : sta $64
        jsr SetPixel

        ; Fehlerterm aktualisieren
        ; (vollständige Implementierung benötigt vorzeichenbehaftete Arithmetik)
        inc cx
        lda err
        clc
        adc dy
        sta err
        cmp dx
        bcc @no_step_y
        inc cy
        lda err : sec : sbc dx : sta err

@no_step_y:
        dey
        bne @loop
        rts

dx: !byte 0
dy: !byte 0
cx: !byte 0
cy: !byte 0
err: !byte 0
step_count: !byte 0

5. FLI — Flexible Line Interpretation

FLI ist eine der faszinierendsten Techniken im C64-Demo-Coding. Sie nutzt ein bestimmtes Verhalten des VIC-II, um die Farbeinschränkungen des Bitmap-Modus dramatisch zu erweitern.

5.1 Das Grundproblem

Im Hires-Bitmap-Modus sind pro 8×8-Pixel-Zelle nur 2 Farben möglich. Das bedeutet: 8 aufeinanderfolgende Rasterzeilen teilen sich dasselbe Farb-Paar aus dem Screen-RAM. Diese Einschränkung macht naturgetreue Bilder schwierig.

5.2 Die FLI-Lösung

FLI umgeht dies durch einen Raster-IRQ auf jeder Bildzeile. Bei jedem IRQ wird der Screen-RAM-Pointer im VIC-II-Register $D018 auf einen anderen Block geändert. So sieht der VIC-II für jede der 8 Rasterzeilen innerhalb einer Zelle einen anderen Screen-RAM-Block — und damit andere Farben!

; FLI-IRQ: wird auf jeder Rasterzeile ausgelöst
; Ändert Screen-RAM-Block per Zeile für mehr Farben

fli_line: !byte 0   ; Aktuelle Rasterzeile (0-199)

FLI_IRQ:
        lda $d012           ; Aktuelle Rasterzeile
        sec
        sbc #51             ; Rasterzeile 51 = erste Bildzeile
        and #$07            ; Welche Zeile in der aktuellen 8er-Gruppe? (0-7)

        ; Screen-RAM-Block für diese Zeile wählen
        ; Block 0 bei $4000, Block 1 bei $4400, ..., Block 7 bei $5C00 (in Bank 1)
        ; $D018 Bits 7-4: Offset / $0400
        ; $4000 in Bank 1: Offset $0000 → Wert 0
        ; $4400 in Bank 1: Offset $0400 → Wert 1
        asl
        asl
        asl
        asl             ; * 16 (= Shift für $D018 Bits 7-4)
        ora #%00001000  ; Bitmap bei $2000 in Bank 1: Bit 3 = 1
        sta $d018       ; Screen-RAM-Block wechseln!

        ; IRQ auf nächste Rasterzeile setzen
        inc $d012

        ; IRQ quittieren
        lda #$01
        sta $d019
        rti
⚠ FLI-Artefakt: "FLI-Bugs"

FLI erzeugt ein charakteristisches 3-Pixel-breites Streifenartefakt am linken Bildschirmrand. Dies entsteht, weil der VIC-II 3 Zyklen benötigt, um den neuen Screen-RAM-Block nach dem $D018-Schreiben zu aktivieren. Professionelle FLI-Implementierungen verstecken diesen Bereich hinter einem Sprite-Vorhang.

6. Grafikkonvertierung und Tools

6.1 Empfohlene Konvertierungs-Tools

  • Timanthes: Vollständiger C64-Grafikeditor für Multicolor, Hires, Sprites
  • Pixcen: Moderner C64-Pixeleditor mit Palette-Einschränkungen
  • VICE Screenshot: Direkt aus dem Emulator als .koala speichern
  • img2c64: Kommandozeilen-Konverter für PC-Bilder → C64-Formate

6.2 Farbpaletten-Beschränkungen

Bei der Bildkonvertierung gibt es harte Einschränkungen:

  • Nur 16 C64-Farben insgesamt (keine Truecolor)
  • Im Multicolor-Bitmap: Pro 4×8-Pixel-Zelle max. 4 Farben, davon 1 global
  • Die C64-Farbpalette ist sehr unregelmäßig verteilt — kein gradueller Verlauf
  • Dithering (abwechselnde Pixel verschiedener Farben) kann optischen Farbmix simulieren

✍ Übungen

Übung 5.1 — Hires-Bitmap-Linien

Implementiere eine vollständige DrawLine-Routine mit dem Bresenham-Algorithmus. Zeichne ein X-Muster (beide Diagonalen des Bildschirms) und ein Rechteck in der Mitte. Verwende weiße Linien auf schwarzem Hintergrund.

Übung 5.2 — Multicolor-Bitmap

Aktiviere den Multicolor-Bitmap-Modus. Fülle den gesamten Bildschirm mit einem geometrischen Muster: Abwechselnd %01 und %10 Pixel (im Schachbrettmuster) mit kontrastierenden Farben in Screen-RAM und $D021/$D022.

Übung 5.3 — Screen-RAM-Farbgebung

Erstelle im Hires-Bitmap-Modus ein Farbverlaufsmuster: Jede Spalte der 40 Zellenspalten erhält ein anderes Farbpaar im Screen-RAM (von Farbe 0 aufwärts zyklisch). Male dann ein einfaches geometrisches Motiv in die Bitmap.

Übung 5.4 — Kreis zeichnen

Implementiere eine DrawCircle(cx, cy, r)-Routine mit dem Midpoint-Circle-Algorithmus. Zeichne konzentrische Kreise auf dem Hires-Bitmap-Bildschirm. Tipp: Nutze die 8-Wege-Symmetrie des Kreises für maximale Effizienz.
