Anhang I

ACME Assembler — Vollständige Referenz

Version 0.97+6510 / 6502 / 65c02Cross-Assembler · alle Direktiven

ACME (Any CPU Machine Assembler) ist ein freier Cross-Assembler, der auf dem PC lauffähig ist und .prg-Dateien für den C64 erzeugt. Neben dem vollständigen 6510-Befehlssatz bietet ACME ein mächtiges Direktiven-System für Datendefinition, Makros, konditionale Assemblierung und Dateieinbindung. Diese Referenz listet alle praxisrelevanten Direktiven, Ausdrücke und Kommandozeilenoptionen.

1. Datendefinitions-Direktiven

1.1 Zahlendaten definieren

DirektiveGrößeBeschreibungBeispiel
!byte1 Byte8-Bit-Wert(e), kommagetrennt!byte $00, $FF, 42
!word2 Bytes16-Bit, Little-Endian!word $1234 → $34, $12
!243 Bytes24-Bit, Little-Endian!24 $123456
!324 Bytes32-Bit, Little-Endian!32 $12345678
!fillN BytesN Bytes mit Wert (Standard: 0)!fill 256, $FF
!alignvariabelAuffüllen bis alignment!align $FF, 0 → auf 256er-Grenze
 
; Beispiele
spritedata: !byte $18,$3C,$7E,$FF,$FF,$FF,$7E,$3C  ; Sprite-Zeile
sinuswert:  !word sin_table_base          ; 16-Bit-Zeiger
padding:    !fill 64, 0                    ; 64 Nullbytes

; Auf Page-Grenze ausrichten (wichtig für Sinus-Tabellen!)
            !align $FF, 0              ; Fülle auf nächste 256er-Grenze
sin_table:  !fill 256, 0               ; Platz für 256 Bytes — page-aligned!

; Text-Direktiven

!binary "charset.bin", 2048      ; nur die ersten 2048 Bytes
!binary "music.sid", 0, $7E      ; SID-Datei ab Offset $7E

3. Makros

Makros ermöglichen parametrisierte, wiederverwendbare Code-Blöcke. Im Gegensatz zu Subroutinen werden Makros bei jedem Aufruf in den Code eingebettet (Inline-Expansion) — keine JSR/RTS-Overhead, aber größerer Code.

3.1 Makro-Definition und Aufruf

; Makro-Definition: !macro name .param1, .param2 { ... }
!macro set_color .addr, .color {
    lda #.color
    sta .addr
}

; Aufruf: +macroname param1, param2
    +set_color $D020, 0          ; Rahmenfarbe = schwarz
    +set_color $D021, 6          ; Hintergrundfarbe = blau

; Makro mit Berechnungen
!macro sprite_pos .num, .x, .y {
    lda #.x & $FF
    sta $D000 + .num * 2       ; X-Position-Register
    lda #.y
    sta $D001 + .num * 2       ; Y-Position-Register
    ; X-MSB (Bit 9) behandeln:
    !if .x > 255 {
        lda $D010
        ora #1 << .num
        sta $D010
    } else {
        lda $D010
        and #<(~(1 << .num))
        sta $D010
    }
}

    +sprite_pos 0, 160, 100     ; Sprite 0 auf (160, 100)
    +sprite_pos 1, 300, 80      ; Sprite 1 auf (300, 80) — X > 255!

3.2 Häufige Assembler-Aufgaben als Makros

; 16-Bit-Wert in Zero-Page laden
!macro load16 .zp, .value {
    lda #<.value
    sta .zp
    lda #>.value
    sta .zp + 1
}

; 16-Bit-Wert inkrementieren
!macro inc16 .zp {
    inc .zp
    bne +
    inc .zp + 1
+
}

; Register sichern und wiederherstellen
!macro push_regs {
    pha
    txa : pha
    tya : pha
}
!macro pop_regs {
    pla : tay
    pla : tax
    pla
}

; VBlank warten
!macro wait_vblank {
-   lda $D011
    bpl -               ; warten bis Bit 7 gesetzt (Rasterzeile >= 256)
-   lda $D011
    bmi -               ; warten bis Bit 7 wieder 0
}

; Verwendung:
    +load16 $FB, my_data_table
    +wait_vblank
    +push_regs
    jsr some_subroutine
    +pop_regs
Makros vs. Subroutinen Makros sind gut für kurze Sequenzen (2–6 Befehle) die keine Zyklen für JSR/RTS verschwenden sollen. Für Routinen die mehr als 10 Zeilen haben oder von vielen Stellen aufgerufen werden: Subroutine bevorzugen. Makros erhöhen die Codegröße proportional zur Aufrufanzahl.

4. Konditionale Assemblierung

4.1 !if / !ifdef / !ifndef

; Einfaches !if mit optionalem else
!if DEBUG = 1 {
    lda #14              ; Hellblau — nur in Debug-Builds
    sta $D020
}

; PAL/NTSC-Konfiguration
PAL  = 1                 ; Konstante für Plattform-Auswahl

!if PAL = 1 {
    RASTERS_PER_FRAME = 312
    CYCLES_PER_LINE   = 63
} else {
    RASTERS_PER_FRAME = 263
    CYCLES_PER_LINE   = 65
}

; Zur Assemblier-Zeit mit !warn prüfen
!if RASTERS_PER_FRAME > 400 {
    !error "Ungültiger RASTERS_PER_FRAME-Wert"
}

; Label-Existenz prüfen
!ifdef USE_KRILL_LOADER {
    jsr LOADER_LOAD
} else {
    jsr kernal_load_file
}

; !ifndef: wenn Label NICHT definiert
!ifndef MUSIC_ADDR {
    MUSIC_ADDR = $E000   ; Standard-Wert wenn nicht gesetzt
}

4.2 Build-Konfiguration mit Konstanten

; config.asm — einmal definieren, überall !source einbinden
DEBUG        = 0         ; 1 = Debug-Ausgaben aktiv
PAL          = 1         ; 0 = NTSC
USE_FASTLOAD = 1         ; 0 = KERNAL-Load
LEVELS       = 8         ; Anzahl Spiellevels
MUSIC_TRACKS = 3         ; Anzahl Musikstücke

; In main.asm:
!source "config.asm"
!if DEBUG = 1 { !warn "Debug-Build aktiv!" }
!if USE_FASTLOAD = 1 { !source "krill/loader.asm" }

; !warn gibt Meldung zur Assemblierzeit aus (kein Fehler)
!warn "Assemblierung für PAL, ", RASTERS_PER_FRAME, " Rasterzeilen"

5. Schleifen und PC-Kontrolle

5.1 !for — Schleifen zur Assemblierzeit

; !for var, count { ... } — count Iterationen, var = 0..count-1

; Sinus-Tabelle generieren (256 Werte, Amplitude ±127)
; ACME unterstützt sin(), cos() als eingebaute Funktionen
sin_table:
!for i, 256 {
    !byte <(sin(i * 3.14159265 * 2 / 256) * 127 + 128)
}

; Lookup-Tabelle: Quadrate 0–255
sqr_table:
!for i, 256 {
    !byte <(i * i / 256)
}

; Sprite-Daten für 8 identische Sprites generieren
sprite_pointers:
!for i, 8 {
    !byte (sprite_data - $0000) / 64   ; Sprite-Pointer
}

; Code unrollen: 8× denselben Befehl ohne Schleife
!for i, 8 {
    !byte $95, $00 + i   ; STA $00+i,X als Rohdaten (Beispiel)
}

5.2 !pseudopc — Code für anderen Speicherort assemblieren

; !pseudopc addr { ... } — Code wird so assembliert als wäre er bei 'addr',
; liegt aber an der aktuellen Assemblierposition (wichtig für Relozierung)

; Drive-Code für 1541: wird an PC-Position gespeichert,
; aber so assembliert als würde er im 1541-RAM bei $0300 laufen
drive_code_start:
!pseudopc $0300 {
drive_entry:
    lda #$05            ; 1541-RAM wird adressiert
    sta $1C05           ; 1541-VIA-Register (nur in 1541 gültig!)
    rts
}
drive_code_end:
DRIVE_CODE_LEN = drive_code_end - drive_code_start

; Code in ZP-Puffer laden und von dort ausführen
zp_temp = $FA
zp_routine_start:
!pseudopc zp_temp {
    inc $D020           ; Rahmenfarbe blinken — wird in ZP als selbst-modifizierender Code laufen
    rts
}
zp_routine_end:

; Lade-Sequenz: Code aus ROM/ROM-Image in ZP kopieren
copy_to_zp:
    ldx #zp_routine_end - zp_routine_start - 1
-   lda zp_routine_start,x
    sta zp_temp,x
    dex
    bpl -
    jsr zp_temp         ; Code jetzt in ZP ausführen
    rts

5.3 !to und Mehrfach-Output

; Output-Datei festlegen (normalerweise über Kommandozeile)
!to "mygame.prg", cbm    ; CBM-Format (2-Byte-Adress-Header)
!to "data.bin", plain    ; kein Header, reine Binärdaten

; Startadresse setzen
* = $0801                ; BASIC-Startadresse
!byte $0B,$08,$01,$00,$9E,$32,$30,$36,$31,$00,$00,$00
                         ; BASIC-Stub "10 SYS 2049" ($0801)
* = $080D
    ; eigentlicher Code ab hier

; Quellcode einbinden (wird vollständig assembliert)

; Hinweis: !source sucht relativ zur assemblierenden Datei.
; Absolute Pfade sind möglich, aber nicht portabel.

8. Vollständige Makro-Bibliothek

Diese Makro-Sammlung kann direkt per !source "macros.asm" in jedes Projekt eingebunden werden:

; macros.asm — C64 ACME Makro-Bibliothek ; Verwendung: !source "macros.asm" ; ─── 16-Bit-Operationen ────────────────────────────────────── !macro load16 .zp, .val { lda #<.val : sta .zp lda #>.val : sta .zp+1 } !macro store16 .zp, .addr { lda .zp : sta .addr lda .zp+1 : sta .addr+1 } !macro inc16 .zp { inc .zp : bne + : inc .zp+1 + } !macro dec16 .zp { lda .zp : bne + : dec .zp+1 + dec .zp } ; ─── Register ──────────────────────────────────────────────── !macro push_regs { pha : txa : pha : tya : pha } !macro pop_regs { pla : tay : pla : tax : pla } ; ─── Video ─────────────────────────────────────────────────── !macro wait_vblank { - lda $D011 : bpl - ; warte auf Rasterzeile >= 256 - lda $D011 : bmi - ; warte auf Rasterzeile < 256 } !macro set_raster .line { lda $D011 !if .line > 255 { ora #$80 } else { and #$7F } sta $D011 lda #<.line : sta $D012 } !macro set_border .color { lda #.color : sta $D020 } !macro set_bg .color { lda #.color : sta $D021 } ; ─── Sprites ───────────────────────────────────────────────── !macro sprite_on .num { lda $D015 : ora #1<<.num : sta $D015 } !macro sprite_off .num { lda $D015 : and #~(1<<.num) : sta $D015 } !macro set_sprite_ptr .num, .frame { lda #(.frame - $0000) / 64 sta $07F8 + .num } ; ─── Arithmetik ────────────────────────────────────────────── !macro mul2 { asl } !macro div2 { lsr } !macro clamp_byte .max { cmp #.max+1 : bcc + : lda #.max + } ; ─── Timing ────────────────────────────────────────────────── !macro nop2 { nop : nop } !macro nop4 { nop : nop : nop : nop } !macro delay .cycles { ; .cycles muss gerade sein (jedes NOP = 2 Zyklen) !for i, .cycles/2 { nop } } ; ─── Tabellen-Generierung zur Assemblierzeit ───────────────── ; Verwendung: !align $FF,0 : sin_table: !for i,256 { !byte <(sin(i*2*3.14159/256)*127+128) }

Skill-Benchmark: ACME-Referenz

  • ☐Du definierst Byte-Arrays, Wörter und Text-Strings mit den passenden ACME-Direktiven
  • ☐Du bindest externe Binärdateien (Sprites, SID-Musik) mit !binary korrekt ein
  • ☐Du nutzt globale Labels, lokale .labels und anonyme +/- Sprünge situationsgerecht
  • ☐Du schreibst ein parametrisiertes Makro mit mindestens 2 Parametern und rufst es auf
  • ☐Du verwendest !if PAL/!if NTSC für plattformübergreifenden Code mit build-time Konfiguration
  • ☐Du generierst eine Sinus-Tabelle mit !for und der eingebauten sin()-Funktion
  • ☐Du nutzt !pseudopc um Drive-Code für die 1541 zu assemblieren
  • ☐Du baust ein Makefile das ACME mit --depfile und --symbollist aufruft
