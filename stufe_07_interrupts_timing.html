Interrupts & Timing

Fortgeschritten 🕑 ca. 8 Stunden Voraussetzungen: Stufen 0–6

🎯 Lernziele dieser Stufe

  • Das vollständige Interrupt-System des C64 (IRQ/NMI) verstehen
  • CIA1 und CIA2 für Tastatur, Joystick und Timer konfigurieren
  • Raster-IRQs präzise implementieren und den Standard-KERNAL-IRQ ersetzen
  • Die Double-IRQ-Technik für stabile Rasterzeilen einsetzen
  • CIA-Timer für musik-synchrone 50 Hz Aufrufe konfigurieren
  • Mehrere IRQs in einer Kette verbinden (Multi-Raster-IRQ)

1. Das Interrupt-System des 6510

Interrupts sind die Grundlage der gesamten Demo- und Spieleprogrammierung auf dem C64. Ohne sie könnten keine Raster-Effekte, keine präzisen Timings und keine gleichzeitigen Aufgaben (Grafik + Musik) realisiert werden. Zu verstehen, wie der C64 Interrupts verarbeitet, ist daher unverzichtbar.

1.1 IRQ — Maskierbarer Interrupt

Der IRQ (Interrupt Request) ist der häufig verwendete Interrupt-Typ. Er kann durch das I-Flag im Status-Register gesperrt (SEI) und erlaubt (CLI) werden. Mögliche Auslöser:

  • VIC-II Raster-Interrupt (wenn Rasterzeile erreicht)
  • CIA1 Timer A oder B Unterlauf
  • CIA1 FLAG-Pin (serieller Bus)

IRQ-Ablauf Schritt für Schritt:

  1. CPU beendet den aktuellen Befehl (1–7 Zyklen Latenz)
  2. CPU prüft I-Flag: gesetzt → IRQ ignoriert; gelöscht → weiter
  3. PC und Status-Register (P) werden auf den Stack gerettet (3 Bytes)
  4. I-Flag automatisch gesetzt (weitere IRQs während Handler gesperrt)
  5. CPU lädt IRQ-Vektor aus $FFFE/$FFFF (ROM)
  6. ROM-Routine springt zu RAM-Vektor $0314/$0315

Die wichtigsten RAM-Vektoren:

AdresseVektor-TypStandard-Ziel
$0314/$0315IRQ-Handler$EA31 (KERNAL-IRQ)
$0316/$0317BRK-Handler$FE66 (KERNAL-BRK)
$0318/$0319NMI-Handler$FE47 (KERNAL-NMI)

1.2 NMI — Nicht-Maskierbarer Interrupt

Der NMI (Non-Maskable Interrupt) kann durch das I-Flag nicht gesperrt werden. Er wird ausgelöst durch:

  • RESTORE-Taste (direkt am NMI-Pin der CPU)
  • CIA2 FLAG-Pin
  • CIA2 Timer (bei entsprechender Konfiguration)

NMI-Vektor: $FFFA/$FFFB (ROM) → $0318/$0319 (RAM).

; RESTORE-Taste abfangen (NMI übernehmen):
        lda #<MyNMIHandler
        sta $0318
        lda #>MyNMIHandler
        sta $0319

MyNMIHandler:
        ; Eigene Aktion bei RESTORE-Taste
        ; z.B.: Spiel pausieren, Menü anzeigen
        lda #$00 : sta $d020    ; Border kurz aufleuchten
        ; Nach Verarbeitung normale Rückkehr:
        rti

2. Die CIA-Chips vollständig

2.1 CIA1 — Tastatur und Joystick ($DC00–$DCFF)

OffsetRegisterFunktion
$DC00PRAPort A: Tastatur-Spalten (Ausgabe), Joystick 2 (Eingabe)
$DC01PRBPort B: Tastatur-Zeilen (Eingabe), Joystick 1 (Eingabe)
$DC02DDRADatenrichtung Port A (Standard: $FF = alles Ausgabe)
$DC03DDRBDatenrichtung Port B (Standard: $00 = alles Eingabe)
$DC04/$DC05TALO/TAHITimer A Zähler (lesen = aktueller Wert, schreiben = Latch)
$DC06/$DC07TBLO/TBHITimer B Zähler
$DC08–$DC0BTODTime of Day Uhr (Zehntel, Sek, Min, Std in BCD)
$DC0CSDRSerielles Datenregister
$DC0DICRInterrupt Control: Lesen = Status (löscht), Schreiben = Maske
$DC0ECRAControl Register A (Timer A steuern)
$DC0FCRBControl Register B

2.2 Tastatur scannen

Die C64-Tastatur ist als 8×8-Matrix aufgebaut: Port A wählt eine Spalte aus (durch Ausgabe einer 0 im entsprechenden Bit), Port B liest dann, welche Zeilen in dieser Spalte gedrückt sind (0 = gedrückt).

; Vollständiger Tastatur-Scan
; Ergebnis in $D4: Matrix-Code (0-63) oder $FF (keine Taste)

ScanKeyboard:
        lda #$ff
        sta $d4         ; Keine Taste gefunden

        lda #%11111110  ; Spalte 0 aktivieren
        ldx #0          ; Spalten-Zähler

@scan_col:
        sta $dc00       ; Spalte setzen
        nop : nop       ; Kurz warten (Stabilisierung)

        lda $dc01       ; Zeilen lesen
        eor #$ff        ; Invertieren (1 = gedrückt)
        beq @next_col   ; Keine Taste in dieser Spalte

        ; Zeile bestimmen (welches Bit gesetzt?)
        ldy #0
@find_row:
        lsr
        bcs @found
        iny
        cpy #8
        bne @find_row
        bne @next_col

@found:
        ; Matrix-Code = Spalte*8 + Zeile
        txa
        asl : asl : asl ; Spalte * 8
        sty $d5
        clc
        adc $d5
        sta $d4         ; Ergebnis speichern

@next_col:
        inx
        cpx #8
        beq @done
        lda ColMasks,x
        jmp @scan_col

@done:
        lda #$ff
        sta $dc00       ; Port A zurücksetzen
        rts

ColMasks: !byte $fe,$fd,$fb,$f7,$ef,$df,$bf,$7f

2.3 CIA2 — VIC-Bank und IEC ($DD00–$DDFF)

OffsetRegisterFunktion
$DD00PRABits 0-1: VIC-Bank (invertiert!); Bit 2: Datasette; Bits 3-5: IEC-Bus
$DD01PRBRS-232, IEC-Bus Status
$DD0DICRCIA2 Interrupt Control (NMI-Quelle!)
 
; VIC-Bank wechseln (CIA2 $DD00 Bits 0-1, invertiert!)
; Bank 0 ($0000): %11 → $DD00 Bits 0-1 = %11 → AND-Maske: ORA $DD00 mit %00000011
; Bank 1 ($4000): %10 → Bits 0-1 = %10
; Bank 2 ($8000): %01 → Bits 0-1 = %01
; Bank 3 ($C000): %00 → Bits 0-1 = %00

SetVICBank1:   ; Auf Bank 1 ($4000-$7FFF) wechseln
        lda $dd00
        and #%11111100  ; Bits 0-1 löschen
        ora #%00000010  ; %10 = Bank 1
        sta $dd00
        rts

3. CIA-Timer konfigurieren

3.1 Timer-Betriebsmodi

Jeder CIA-Chip hat zwei 16-Bit-Timer (A und B). Das Control-Register (CRA/CRB) bestimmt den Betriebsmodus:

BitNameFunktion
0START1 = Timer läuft, 0 = angehalten
3RUNMODE0 = Continuous (automatisch neu laden), 1 = One-Shot (stoppt nach Ablauf)
4LOADSchreiben einer 1 lädt Latch-Wert sofort in Zähler
5INMODE0 = Systemclock, 1 = CNT-Pin als Taktquelle

3.2 Präziser 50 Hz Timer-IRQ

; CIA1 Timer A für exakt 50 Hz (PAL) konfigurieren
; PAL: 985248 Hz, 985248 / 50 = 19704,96 ≈ 19704 Zyklen = $4CC8

InitTimer50Hz:
        SEI
        ; Alle CIA-IRQs deaktivieren
        LDA #$7F
        STA $DC0D       ; CIA1: alle IRQs aus (Bit 7=0 → Bits löschen)
        STA $DD0D       ; CIA2: alle IRQs aus
        LDA $DC0D       ; Pending-Bits löschen (durch Lesen)
        LDA $DD0D

        ; Eigenen IRQ-Vektor setzen
        LDA #<TimerHandler
        STA $0314
        LDA #>TimerHandler
        STA $0315

        ; Timer A Latch auf 19704 ($4CC8) setzen
        LDA #$C8        ; $4CC8 Low-Byte
        STA $DC04       ; Timer A Low (schreibt Latch)
        LDA #$4C        ; $4CC8 High-Byte
        STA $DC05       ; Timer A High (schreibt Latch)

        ; Control Register A: Continuous + Systemclock + sofortiges Laden + Start
        LDA #%00010001  ; Bit 4 (LOAD) + Bit 0 (START)
        STA $DC0E

        ; Timer A IRQ aktivieren
        LDA #%10000001  ; Bit 7=1 (Bits setzen), Bit 0=Timer A
        STA $DC0D

        CLI
        RTS

TimerHandler:
        LDA $DC0D       ; Interrupt-Status lesen (quittiert CIA1-IRQ!)
        ; Eigener Code hier...
        JMP $EA31       ; KERNAL-IRQ für Tastatur etc. aufrufen

4. Raster-IRQ vollständig

4.1 Grundaufbau

Der VIC-II Raster-IRQ ist das wichtigste Werkzeug für Grafikeffekte. Er wird ausgelöst, wenn der Raster-Strahl eine bestimmte Zeile erreicht. Die Ziel-Zeile wird in $D012 (Bits 0–7) und $D011 Bit 7 (Bit 8 für Zeilen >255) gesetzt.

; =====================================================
; Raster-IRQ Setup — vollständige Routine
; Farb-Split: Oberer Bildschirmbereich rot, unterer blau
; =====================================================

IRQ_LINE_1 = 100    ; Wechsel zu Blau
IRQ_LINE_2 = 200    ; Wechsel zurück zu Rot

SetupRasterIRQ:
        SEI
        ; CIA-IRQs deaktivieren
        LDA #$7F : STA $DC0D : STA $DD0D
        LDA $DC0D : LDA $DD0D   ; Pending löschen

        ; IRQ-Vektor auf IRQ1
        LDA #<IRQ1 : STA $0314
        LDA #>IRQ1 : STA $0315

        ; Raster-Zeile für IRQ1 setzen
        LDA #IRQ_LINE_1
        STA $D012
        LDA $D011
        AND #$7F        ; Bit 7 = 0 (Zeile < 256)
        STA $D011

        ; Raster-IRQ aktivieren
        LDA #$01
        STA $D01A

        CLI
        RTS

IRQ1:
        ; Farbe wechseln: Blau
        LDA #6 : STA $D020 : STA $D021

        ; IRQ2 auf Zeile 200 setzen
        LDA #IRQ_LINE_2 : STA $D012
        LDA #<IRQ2 : STA $0314
        LDA #>IRQ2 : STA $0315

        ; IRQ quittieren
        LDA #$01 : STA $D019
        JMP $EA31

IRQ2:
        ; Farbe wechseln: Rot
        LDA #2 : STA $D020 : STA $D021

        ; IRQ1 zurücksetzen
        LDA #IRQ_LINE_1 : STA $D012
        LDA #<IRQ1 : STA $0314
        LDA #>IRQ1 : STA $0315

        LDA #$01 : STA $D019
        JMP $EA31

4.2 Das Jitter-Problem und die Double-IRQ-Lösung

Ein fundamentales Problem beim Raster-IRQ ist der sogenannte Jitter: Der 6510 muss stets den aktuellen Befehl zu Ende ausführen. Da Befehle zwischen 2 und 7 Zyklen dauern, variiert der Zeitpunkt des IRQ-Eintreffens um bis zu 7 Zyklen. Bei Timing-kritischen Effekten wie Farbsplits auf einer einzelnen Rasterzeile erzeugt das ein sichtbares "Zittern".

Die Lösung ist die Double-IRQ-Technik: Zwei aufeinanderfolgende IRQs werden verwendet. Der erste IRQ richtet den zweiten exakt aus; der zweite führt dann den eigentlichen Effekt aus.

; =====================================================
; Stabiler Raster-IRQ (Double-IRQ / Stable Raster)
; Ziel: exakt auf Rasterzeile EFFECT_LINE
; =====================================================

EFFECT_LINE = 120

SetupStableRaster:
        SEI
        LDA #$7F : STA $DC0D : STA $DD0D
        LDA $DC0D : LDA $DD0D

        LDA #<StableIRQ1 : STA $0314
        LDA #>StableIRQ1 : STA $0315

        LDA #(EFFECT_LINE-1) : STA $D012
        LDA $D011 : AND #$7F : STA $D011
        LDA #$01 : STA $D01A
        CLI
        RTS

; IRQ1: Stellt IRQ2 exakt auf die Zielzeile
StableIRQ1:
        LDA #$01 : STA $D019    ; IRQ quittieren

        ; Auf IRQ2 vorbereiten
        LDA #EFFECT_LINE : STA $D012
        LDA #<StableIRQ2 : STA $0314
        LDA #>StableIRQ2 : STA $0315

        ; Timing-Ausgleich: NOP-Padding je nach CPU-Phase
        ; Die genaue Anzahl NOPs hängt vom Assembler-Timing ab
        ; Standard-Technik für sehr präzises Timing:
        BIT $EA31       ; 4 Zyklen (für Timing-Ausgleich)
        RTI

; IRQ2: Effekt läuft hier — exakt auf EFFECT_LINE
StableIRQ2:
        LDA #$01 : STA $D019

        ; EFFEKT (exakt auf der gewünschten Zeile!)
        LDA #5 : STA $D020  ; Grüne Border auf dieser Zeile
        ; (für dauerhaft: hier Farbwechsel für den Rest des Frames)

        ; Zurück zu IRQ1 für nächsten Frame
        LDA #(EFFECT_LINE-1) : STA $D012
        LDA #<StableIRQ1 : STA $0314
        LDA #>StableIRQ1 : STA $0315

        JMP $EA31
💡 Timing-Ausgleich im Detail

Das genaue NOP-Padding in IRQ1 ist abhängig davon, auf welchem Zyklus innerhalb der Rasterzeile der IRQ eintrifft. Für maximale Präzision (±0 Zyklen) muss IRQ1 einen "Wartemechanismus" implementieren: Warten bis ein bekannter Raster-Zyklus-Punkt erreicht ist, dann RTI. In der Praxis nutzen Demo-Coder oft die Technik, INC $D012 zweimal aufzurufen und die genauen Zyklenzahlen zu zählen.

5. Vollständiger Musik-IRQ

; =====================================================
; Kombination: Raster-IRQ für Grafik + Musik-Update
; Musik-Player wird im vertikalen Blankintervall aufgerufen
; =====================================================

MUSIC_LINE = 250    ; Rasterzeile im VBlank-Bereich (PAL)

SetupMusicIRQ:
        SEI
        LDA #$7F : STA $DC0D : STA $DD0D
        LDA $DC0D : LDA $DD0D

        LDA #<MusicIRQ : STA $0314
        LDA #>MusicIRQ : STA $0315

        LDA #MUSIC_LINE : STA $D012
        LDA $D011 : AND #$7F : STA $D011
        LDA #$01 : STA $D01A
        CLI
        RTS

MusicIRQ:
        LDA #$01 : STA $D019    ; Quittieren
        JSR MusicUpdate         ; Musik-Player aufrufen (aus Stufe 6)
        JMP $EA31               ; KERNAL-IRQ weiter

Übung 7.1 — Regenbogen-Border

Implementiere einen IRQ, der auf jeder sichtbaren Rasterzeile die Border-Farbe wechselt. Das Ergebnis soll ein Regenbogen aus allen 16 Farben in der Border sein, der sich jede Sekunde verschiebt (durch einen Zähler, der den Farb-Offset verändert).

Übung 7.2 — Musik + Grafik gleichzeitig

Kombiniere den Musik-Player aus Stufe 6 mit einem Raster-IRQ. Die Musik soll im VBlank-IRQ (Rasterzeile 250+) laufen. Gleichzeitig soll ein zweiter IRQ auf Rasterzeile 100 die Border-Farbe rhythmisch zur Musik ändern.

Übung 7.3 — Stabile Farbsplits

Implementiere 4 stabile Farbsplits auf dem Bildschirm (4 verschiedene Background-Farben in 4 horizontalen Streifen). Verwende die Double-IRQ-Technik für jeden Split. Die Grenzen sollen pixelgenau sein (kein Jitter sichtbar).
