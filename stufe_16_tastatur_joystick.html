1. CIA1-Architektur für Eingabe

Alle Tastatur- und Joystick-Signale des C64 laufen durch den CIA1-Chip (Complex Interface Adapter) bei $DC00. Der CIA1 hat zwei 8-Bit I/O-Ports mit konfigurierbaren Datenrichtungen. Für Eingabegeräte sind vor allem vier Register relevant:

AdresseNameFunktionRichtung
$DC00CIA1 PRA (Port A)Joystick Port 2 + Tastatur Zeilen-SelectOutput (Scan)
$DC01CIA1 PRB (Port B)Joystick Port 1 + Tastatur Spalten-ReadInput (Read)
$DC02CIA1 DDRADatenrichtung Port A (0=Input, 1=Output)Konfig
$DC03CIA1 DDRBDatenrichtung Port B (0=Input, 1=Output)Konfig

Der C64-Startup setzt $DC02 = $FF (Port A = alles Output) und $DC03 = $00 (Port B = alles Input). Das ist die korrekte Konfiguration für Tastatur-Scanning. Für reine Joystick-Abfrage reicht ein einfaches Lesen von $DC00 und $DC01, ohne die Richtungsregister anzufassen.

Gemeinsame Leitungen
Die Joystick-Ports teilen Leitungen mit der Tastaturmatrix. Port 2 liegt auf $DC00 (Port A), Port 1 auf $DC01 (Port B). Ein gedrückter Joystick-Button kann daher wie eine gedrückte Taste erscheinen — und umgekehrt. Wer Joystick und Tastatur gleichzeitig nutzt, muss das im Code berücksichtigen.

1.1 Initialisierung

Für den häufigsten Anwendungsfall — Joystick Port 2 + Tastatur — muss der CIA1 korrekt konfiguriert sein:

; CIA1 für Tastatur-Scan + Joystick initialisieren
; Dies ist der C64-Standardzustand nach dem Reset
init_input:
    lda #$FF
    sta $DC02           ; DDRA: Port A = Output (Zeilen-Select)
    lda #$00
    sta $DC03           ; DDRB: Port B = Input  (Spalten-Read + Joy1)
    lda #$FF
    sta $DC00           ; Alle Zeilen abwählen (High = nicht selektiert)
    rts

2. Joystick-Abfrage

Der C64 hat zwei 9-Pin-Joystick-Ports. Die Richtungs- und Feuer-Signale sind aktiv Low: 0 bedeutet gedrückt, 1 bedeutet losgelassen. Das ist ein häufiger Anfängerfehler — die Bits müssen invertiert oder mit AND-Masken ausgewertet werden.

Port 2 — $DC00 Bits 0–4

Joystick Port 2 wird von fast allen C64-Spielen genutzt, weil er keine Konflikte mit der Tastatur-Schaltmatrix verursacht (Port A ist Output).

Port 1 — $DC01 Bits 0–4

Joystick Port 1 teilt Leitungen mit dem Tastatur-Port B. Beim Scannen beider Ports gleichzeitig können Tastaturanschläge als Joy-1-Bewegungen registriert werden.

BitRichtung$DC00 (Joy 2)$DC01 (Joy 1)Wert wenn gedrückt
0UpBit 0Bit 00
1DownBit 1Bit 10
2LeftBit 2Bit 20
3RightBit 3Bit 30
4FireBit 4Bit 40
5–7—andere Funktionenandere Funktionen1

2.1 Basis-Lesefunktion

; Joystick Port 2 lesen — gibt invertierten Wert zurück
; (0 = gedrückt → nach EOR: 1 = gedrückt, intuitiver)
read_joy2:
    lda $DC00           ; Port A lesen (Joystick Port 2)
    eor #$FF            ; Invertieren: 1=gedrückt, 0=losgelassen
    and #$1F            ; Nur Bits 0–4 (Up/Down/Left/Right/Fire)
    rts                 ; A enthält Joystick-Zustand

; Auswertung nach read_joy2:
;   AND #$01 → Up    (Bit 0)
;   AND #$02 → Down  (Bit 1)
;   AND #$04 → Left  (Bit 2)
;   AND #$08 → Right (Bit 3)
;   AND #$10 → Fire  (Bit 4)

; Joystick Port 1 lesen
read_joy1:
    lda $DC01           ; Port B lesen (Joystick Port 1)
    eor #$FF
    and #$1F
    rts
Tipp: Konstanten definieren
Definiere Richtungs-Masken als ACME-Konstanten am Anfang deiner Quelldatei: JOY_UP = $01, JOY_DOWN = $02, JOY_LEFT = $04, JOY_RIGHT = $08, JOY_FIRE = $10. Das macht den Code deutlich lesbarer.

3. Joystick in der Praxis

3.1 Edge Detection — nur beim ersten Frame auslösen

Ohne Edge Detection wird eine Aktion so lange wiederholt, wie der Knopf gedrückt ist — 50 Mal pro Sekunde. Für Sprung, Schuss oder Menü-Auswahl soll aber nur der erste Frame reagieren. Edge Detection speichert den vorherigen Zustand und löst nur bei der positiven Flanke (losgelassen → gedrückt) aus.

; Zero-Page-Variablen
joy_curr = $02          ; aktueller Joystick-Zustand
joy_prev = $03          ; Zustand vom letzten Frame
joy_edge = $04          ; Bits die GERADE neu gedrückt wurden

; Joystick-Update — einmal pro Frame im IRQ oder Game-Loop aufrufen
joy_update:
    lda joy_curr        ; vorherigen Zustand sichern
    sta joy_prev
    lda $DC00
    eor #$FF
    and #$1F
    sta joy_curr        ; aktuellen Zustand speichern
    eor joy_prev        ; XOR: welche Bits haben sich geändert?
    and joy_curr        ; AND: nur Bits die jetzt gedrückt sind (= neue Drücke)
    sta joy_edge        ; Edge-Bits speichern
    rts

; Verwendung im Game-Loop:
check_fire_once:
    lda joy_edge
    and #$10            ; Fire-Button Edge?
    beq .no_fire
    jsr player_shoot    ; Nur einmalig auslösen
.no_fire:
    rts

3.2 Vollständiger Spieler-Handler mit Diagonal und Fire-Repeat

; Joystick-Konstanten
JOY_UP    = $01
JOY_DOWN  = $02
JOY_LEFT  = $04
JOY_RIGHT = $08
JOY_FIRE  = $10

; Zero-Page
joy_curr     = $02
joy_prev     = $03
joy_edge     = $04
fire_repeat  = $05      ; Frame-Zähler für Auto-Repeat des Feuerknopfes
player_vx    = $06      ; Geschwindigkeit X (-1, 0, +1)
player_vy    = $07      ; Geschwindigkeit Y

FIRE_REPEAT_DELAY = 8   ; Frames bis Auto-Repeat beginnt
FIRE_REPEAT_RATE  = 4   ; Frames zwischen wiederholten Schüssen

joy_full_update:
    ; --- Zustand lesen + Edge Detection ---
    lda joy_curr
    sta joy_prev
    lda $DC00
    eor #$FF
    and #$1F
    sta joy_curr
    eor joy_prev
    and joy_curr
    sta joy_edge

    ; --- Bewegung: X-Achse ---
    lda #0
    sta player_vx
    lda joy_curr
    and #JOY_LEFT
    beq .check_right
    lda #$FF             ; -1 (links)
    sta player_vx
.check_right:
    lda joy_curr
    and #JOY_RIGHT
    beq .check_up
    lda #1
    sta player_vx       ; +1 (rechts) — überschreibt links bei Konflikt

    ; --- Bewegung: Y-Achse ---
.check_up:
    lda #0
    sta player_vy
    lda joy_curr
    and #JOY_UP
    beq .check_down
    lda #$FF             ; -1 (oben)
    sta player_vy
.check_down:
    lda joy_curr
    and #JOY_DOWN
    beq .fire_check
    lda #1
    sta player_vy

    ; --- Feuer mit Auto-Repeat ---
.fire_check:
    lda joy_curr
    and #JOY_FIRE
    bne .fire_held
    lda #0               ; Feuer losgelassen → Reset Counter
    sta fire_repeat
    rts

.fire_held:
    lda joy_edge         ; Gerade erst gedrückt?
    and #JOY_FIRE
    bne .fire_now        ; Ja → sofort schießen
    inc fire_repeat
    lda fire_repeat
    cmp #FIRE_REPEAT_DELAY
    bcc .no_fire_yet
    ; Auto-Repeat-Phase
    lda fire_repeat
    sec
    sbc #FIRE_REPEAT_DELAY
    and #(FIRE_REPEAT_RATE-1)  ; Modulo FIRE_REPEAT_RATE (muss Potenz von 2 sein)
    bne .no_fire_yet
.fire_now:
    jsr player_shoot
.no_fire_yet:
    rts

4. Die Tastaturmatrix

Die C64-Tastatur funktioniert nach dem Matrix-Scan-Prinzip: 64 Tasten sind in einem 8×8-Raster aus 8 Zeilen und 8 Spalten verdrahtet. Zum Scannen wird eine Zeile nach der anderen auf Low gezogen (über $DC00), und dann geprüft, welche Spalte in $DC01 auf Low reagiert — das ist die gedrückte Taste.

4.1 Scan-Prinzip

  1. $DC00 = $FE: Zeile 0 aktiv (Bit 0 = 0, alle anderen = 1)
  2. $DC01 lesen: welches Bit ist 0? → das ist die Spalte der gedrückten Taste
  3. $DC00 = $FD: Zeile 1 aktiv, $DC01 lesen …
  4. Alle 8 Zeilen durchlaufen → 8 × 8 = 64 mögliche Tasten erkannt

Zeilen werden durch das Schreiben eines Werts mit einem einzelnen Null-Bit in $DC00 selektiert:

Zeile$DC00 WertBinär
0$FE%11111110
1$FD%11111101
2$FB%11111011
3$F7%11110111
4$EF%11101111
5$DF%11011111
6$BF%10111111
7$7F%01111111

4.2 Vollständige Tastaturmatrix

Die Matrix-Tabelle zeigt für jede Kombination aus Zeile (Wert in $DC00) und Spalte (Bit in $DC01) die entsprechende Taste:

$DC00 → Bit 7Bit 6Bit 5Bit 4 Bit 3Bit 2Bit 1Bit 0
$DC01 Bit: 7654 3210
$FE (Z.0)STOPQC=SPACE2CTRL←1
$FD (Z.1)/↑=RSHIFTHOME;*£
$FB (Z.2),@:.-LP+
$F7 (Z.3)NOKM0JI9
$EF (Z.4)VUHB8GY7
$DF (Z.5)XTFC6DR5
$BF (Z.6)LSHIFTESZ4AW3
$7F (Z.7)CUR↓F5F3F1F7CUR→RETURNDEL

Scan-Code einer Taste = Zeile × 8 + Spalten-Bit-Nummer. DEL hat Scan-Code 0×8+0 = 0, STOP hat 0×8+7 = 7, F1 hat 7×8+4 = 60 usw.

5. Tastatur-Scan-Routinen

5.1 Vollständiger Matrix-Scan

; key_state: 64-Byte-Array, 1=gedrückt, 0=losgelassen
; key_prev:  vorheriger Zustand (für Edge Detection)
; Beide im RAM (nicht ZP nötig, Zugriff ist unkritisch)
key_state  = $0340       ; 64 Bytes
key_prev   = $0380       ; 64 Bytes (für Edge Detection)

; Row-Select-Tabelle: 8 Werte für $DC00
row_mask:
    !byte $FE, $FD, $FB, $F7, $EF, $DF, $BF, $7F

scan_keyboard:
    sei                  ; IRQ sperren während Scan
    ldx #0               ; Zeilen-Index
.row_loop:
    ; Vorherigen Zustand dieser Zeile sichern
    ldy #7
.save_prev:
    lda key_state,x      ; Hinweis: x+y wäre ideal, aber X ist Zeile×8
    sta key_prev,x
    inx
    dey
    bpl .save_prev
    txa
    sec
    sbc #8
    tax                  ; X zurück auf Zeilenanfang

    ; Zeile auswählen
    lda row_mask,x       ; Falsch: X ist hier Zeilen-Idx 0–7, nicht ×8
    ; Korrekte Implementierung mit separatem Zähler:
    rts                  ; (vereinfacht — siehe vollständige Version unten)

; ── Sauberere Implementierung ───────────────────────────────────
scan_keys_full:
    sei
    ldx #7              ; Zeilen-Index 7..0
.row:
    lda row_mask,x
    sta $DC00            ; Zeile auswählen
    nop                  ; kurz warten (Signal stabilisiert sich)
    lda $DC01            ; Spalten lesen
    eor #$FF             ; Invertieren: 1=gedrückt
    sta $02              ; zwischenspeichern
    ldy #7              ; Spalten-Index 7..0
.col:
    ; Scan-Code = Zeile×8 + Spalte
    txa
    asl
    asl
    asl                  ; ×8 → Offset für diese Zeile
    sty $03
    clc
    adc $03              ; + Spalte
    tay                  ; Y = Scan-Code (Index ins key_state-Array)
    lda $02
    and col_bit_mask,y   ; Spalten-Bit isolieren
    beq .not_pressed
    lda #1
    bne .store
.not_pressed:
    lda #0
.store:
    sta key_state,y
    ldy $03
    dey
    bpl .col
    dex
    bpl .row
    lda #$FF
    sta $DC00            ; Alle Zeilen wieder abwählen
    cli
    rts

col_bit_mask: !byte $01,$02,$04,$08,$10,$20,$40,$80

5.2 Debounce und Key-Repeat

Entprellen (Debounce) verhindert, dass mechanisches Prellen einer Taste als mehrere Drücke gewertet wird. Eine einfache Methode: eine Taste gilt erst als stabil gedrückt, wenn sie in zwei aufeinanderfolgenden Frames als gedrückt gelesen wird.

; Einfaches 2-Frame-Debounce für eine einzelne Taste
; key_raw: direkt vom Scan, key_db: entprellter Zustand
key_raw = $06
key_db  = $07

debounce_update:
    lda key_raw         ; neuer roher Wert
    and key_db_prev      ; Taste muss in BEIDEN Frames gedrückt sein
    sta key_db           ; entprellter Wert
    lda key_raw
    sta key_db_prev      ; aktuell für nächsten Frame sichern
    rts

; Key-Repeat: erst nach REPEAT_DELAY Frames, dann alle REPEAT_RATE Frames
key_repeat_cnt = $08    ; Frame-Zähler
REPEAT_DELAY = 20       ; 20 Frames ≈ 400 ms bei 50 Hz
REPEAT_RATE  = 3        ; alle 3 Frames ein Repeat ≈ 60 ms

key_repeat_tick:
    lda key_db           ; Taste gedrückt?
    beq .reset_cnt
    inc key_repeat_cnt
    lda key_repeat_cnt
    cmp #REPEAT_DELAY
    bcc .no_repeat
    lda key_repeat_cnt
    sec
    sbc #REPEAT_DELAY
    ; Modulo REPEAT_RATE (= 3 → nicht Potenz von 2, daher CMP)
    cmp #REPEAT_RATE
    bne .no_repeat
    lda #REPEAT_DELAY    ; Zähler auf Delay-Wert zurücksetzen
    sta key_repeat_cnt
    jsr handle_repeated_key
    rts
.no_repeat:
    rts
.reset_cnt:
    lda #0
    sta key_repeat_cnt
    rts

6. Mehrere Tasten gleichzeitig

6.1 Das Ghosting-Problem

Bei mechanischen Matrix-Tastaturen ohne Dioden (wie dem C64) können bei drei gleichzeitig gedrückten Tasten Geistertasten (Ghosts) entstehen. Wenn z.B. A, B und C gedrückt sind, wobei A+B in einer Zeile und A+C in einer Spalte liegen, erscheint die vierte Ecke der "Box" auch als gedrückt.

Für Spielereingaben ist das in der Praxis selten ein Problem, weil:

  • Joystick + Feuer werden über $DC00/$DC01 Bits 0–4 abgefragt — dort kein Ghosting
  • Die meisten Spiele nur 1–2 Tasten gleichzeitig benötigen (Cursors + SPACE)
  • Kritische Kombinationen (Shift + Buchstabe) liegen so in der Matrix, dass sie sicher funktionieren

6.2 Sichere Tastenkombinationen

Folgende Kombinationen funktionieren auf jedem C64 zuverlässig gleichzeitig:

  • Cursor-Tasten (alle vier) + SPACE oder RETURN
  • LSHIFT + beliebige Buchstabentaste
  • Joystick-Richtungen (aus $DC00) + Tastatur (weitgehend unabhängig)
Achtung: Joystick Port 1 + Tastatur
Joystick Port 1 liegt auf $DC01 (Port B) — demselben Register das für das Lesen der Tastaturmatrix-Spalten verwendet wird. Wenn Port 1 einen Joystick hat und gleichzeitig die Tastatur gescannt wird, können Joystick-Bewegungen als Tastenanschläge erscheinen. Für die meisten Spiele deshalb: Joystick Port 2 bevorzugen.

6.3 Key-State-Array für mehrere Tasten

Das Key-State-Array aus Abschnitt 5.1 (64 Bytes, ein Byte pro Scan-Code) erlaubt das gleichzeitige Verfolgen beliebig vieler Tasten. Jedes Byte ist 0 (losgelassen) oder 1 (gedrückt). Abfrage einer einzelnen Taste:

; Scan-Code-Konstanten für häufige Tasten
KEY_DEL    = 0
KEY_RETURN = 1
KEY_F1     = 4      ; Zeile 0, Spalte 4 → 0*8+4 = 4
KEY_F3     = 5      ; Zeile 0, Spalte 5 → 0*8+5 = 5
KEY_F5     = 6      ; Zeile 0, Spalte 6 → 0*8+6 = 6
KEY_F7     = 3      ; Zeile 0, Spalte 3 → 0*8+3 = 3
KEY_SPACE  = 60     ; Zeile 7, Spalte 4 → 7*8+4 = 60
KEY_LSHIFT = 48     ; Zeile 6, Spalte 0 → 6*8+0 = 48
KEY_STOP   = 7      ; Zeile 0, Spalte 7

; Prüfen ob RETURN gedrückt:
is_return_pressed:
    lda key_state + KEY_RETURN
    rts                  ; A ≠ 0 → gedrückt

; Prüfen ob SHIFT + RETURN gedrückt (gleichzeitig):
is_shift_return:
    lda key_state + KEY_RETURN
    beq .no
    lda key_state + KEY_LSHIFT
    rts                  ; A ≠ 0 → Shift+Return gedrückt
.no:
    lda #0
    rts

7.1 GETIN für einfache Eingaben

GETIN liest ein Zeichen aus dem KERNAL-Tastaturpuffer. Ist der Puffer leer, kommt A=0 zurück — ideal für Non-Blocking-Polling im Game-Loop:

; Non-blocking Tastenabfrage mit GETIN
poll_key:
    jsr $FFE4            ; GETIN
    tax                  ; Taste in X sichern
    beq .no_key          ; A=0 → kein Zeichen
    ; A enthält PETSCII-Code der Taste
    cmp #$51             ; 'Q' in PETSCII
    beq .quit
    cmp #$11             ; Cursor Down
    beq .cursor_down
.no_key:
    rts

8. Praxisbeispiele

8.1 Menü-Navigation

; Menü-Steuerung: Cursor Up/Down + RETURN
; Verwendet GETIN für einfache, robuste Implementierung

menu_cursor  = $C0       ; aktuelle Menüposition (0-basiert)
MENU_ITEMS   = 4         ; Anzahl Menüeinträge

menu_handle_input:
    jsr $FFE4            ; GETIN
    beq .done

    cmp #$11             ; Cursor Down (PETSCII $11)
    bne .check_up
    inc menu_cursor
    lda menu_cursor
    cmp #MENU_ITEMS
    bcc .draw
    lda #0               ; Wrap-around
    sta menu_cursor
    bne .draw

.check_up:
    cmp #$91             ; Cursor Up (PETSCII $91)
    bne .check_fire
    lda menu_cursor
    beq .wrap_up
    dec menu_cursor
    bne .draw
.wrap_up:
    lda #(MENU_ITEMS-1)
    sta menu_cursor
    bne .draw

.check_fire:
    cmp #$0D             ; RETURN (PETSCII $0D)
    bne .check_joy_fire
    jsr menu_select_item ; Auswahl bestätigen
    rts

.check_joy_fire:
    lda $DC00
    and #$10             ; Joy2 Fire (Bit 4, active low)
    bne .done            ; Bit gesetzt = nicht gedrückt
    jsr menu_select_item
    rts
.draw:
    jsr menu_draw_cursor
.done:
    rts

8.2 High-Score-Namenseingabe

; High-Score-Namenseingabe: bis zu 8 Zeichen, A–Z + RETURN
; name_buf: 8-Byte-Puffer, name_len: aktuelle Länge

name_buf = $C0           ; 8 Bytes
name_len = $C8           ; aktuelle Länge 0–8
NAME_MAX = 8

hiscore_input_loop:
.poll:
    jsr $FFE4            ; GETIN (wartet nicht, Non-Blocking)
    beq .poll            ; kein Zeichen → weiter warten

    cmp #$0D             ; RETURN → fertig
    beq .done

    cmp #$14             ; DEL → letztes Zeichen entfernen
    bne .check_letter
    lda name_len
    beq .poll            ; schon leer
    dec name_len
    jsr hiscore_draw
    jmp .poll

.check_letter:
    ; Nur A–Z (PETSCII Großbuchstaben $41–$5A) akzeptieren
    cmp #$41
    bcc .poll            ; kleiner als 'A' → ignorieren
    cmp #$5B
    bcs .poll            ; größer als 'Z' → ignorieren
    ldx name_len
    cpx #NAME_MAX
    bcs .poll            ; Puffer voll
    sta name_buf,x       ; Zeichen speichern
    inc name_len
    jsr hiscore_draw
    jmp .poll

.done:
    lda name_len
    beq .poll            ; leerer Name nicht erlaubt
    rts

8.3 Vollständiger Plattformer-Input

Kombiniert Joystick Port 2 mit Edge Detection für Sprung — der klassische Anwendungsfall:

; Platform-Steuerung: Joystick Port 2
; Links/Rechts → Bewegung, Up/Fire → Sprung (Edge-Triggered)

joy2_state = $02
joy2_edge  = $03
joy2_prev  = $04
on_ground  = $05         ; 1 = Spieler steht auf Boden

player_update_input:
    ; Joystick lesen + Edge Detection
    lda joy2_state
    sta joy2_prev
    lda $DC00
    eor #$1F             ; Bits 0–4 invertieren
    and #$1F
    sta joy2_state
    eor joy2_prev
    and joy2_state
    sta joy2_edge

    ; Horizontale Bewegung (gehalten, kein Edge nötig)
    lda joy2_state
    and #$04             ; Links
    beq .chk_right
    jsr player_move_left
.chk_right:
    lda joy2_state
    and #$08             ; Rechts
    beq .chk_jump
    jsr player_move_right

    ; Sprung: nur bei Edge, nur wenn auf dem Boden
.chk_jump:
    lda on_ground
    beq .done            ; in der Luft → kein Sprung
    lda joy2_edge
    and #$11             ; Up (Bit 0) ODER Fire (Bit 4)
    beq .done
    jsr player_jump      ; Sprung einleiten
.done:
    rts

Übungsaufgaben

Übung 16.1 — Joystick-Tester

Schreibe ein Programm das beide Joystick-Ports liest und den Zustand auf dem Bildschirm anzeigt: Pfeile für die Richtungen (↑↓←→), ein gefülltes Quadrat für Fire. Zeige beide Ports nebeneinander. Aktualisiere die Anzeige jeden Frame über einen Raster-IRQ. Verwende direkte Bildschirmspeicher-Ausgabe ohne KERNAL.

Übung 16.2 — Tastatur-Monitor
