Anhang H

KERNAL-Routinen-Referenz

$FF81 – $FFF340 RoutinenJump-Tabelle · Aufruf-Konventionen

Das KERNAL ist ein ROM bei $E000–$FFFF, das geräteunabhängige I/O-Funktionen bereitstellt. Die Jump-Tabelle bei $FF81–$FFF3 enthält stabile Einstiegspunkte — immer über die Jump-Tabellen-Adressen aufrufen, niemals interne KERNAL-Adressen direkt anspringen. Diese internen Adressen können sich zwischen ROM-Revisionen ändern, die Jump-Tabelle bleibt immer gleich. Die Statusvariable STATUS bei $90 wird nach jeder I/O-Operation aktualisiert und gibt Auskunft über Erfolg oder Fehler. Aufruf aller Routinen mit JSR $FFxx.

KERNAL nicht erreichbar bei deaktiviertem I/O Der Bereich $E000–$FFFF muss als ROM eingeblendet sein. Sicherheitsprüfung: LDA $01 / AND #$07 muss $05, $06 oder $07 ergeben. Für KERNAL-Zugriff muss Bit 1 gesetzt sein (Standard: $01 = $37). Bei Maschinenprogrammen, die I/O deaktivieren, vor jedem KERNAL-Aufruf LDA #$37 / STA $01 setzen.
Interrupt-Vektoren ($0314–$0319) $0314/$0315  CINV   IRQ-Vektor (Standard: $EA31 — KERNAL-IRQ)
$0316/$0317  CBINV  BRK-Vektor (Standard: $FE66)
$0318/$0319  NMINV  NMI-Vektor (Standard: $FE47)

Eigene Interrupt-Routinen immer über $0314/$0315 einhängen, NICHT über $FFFE/$FFFF — sonst gehen KERNAL-Funktionen (Tastatur-Scan, TOD-Timer) verloren.

String-Ausgabe mit CHROUT

; String mit CHROUT ausgeben ($FF,$00 = Ende-Markierung)
; Aufruf: LDX #0 / JSR PrintStr (X = Index)

msg:   !pet "HELLO, WORLD!", 13, 0   ; PETSCII, Return am Ende, Null-Terminierung

PrintStr:
       LDA msg,X          ; Zeichen laden
       BEQ .done          ; Null = Ende
       JSR $FFD2          ; CHROUT: Zeichen ausgeben
       INX
       BNE PrintStr       ; nächstes Zeichen (max 255)
.done: RTS

Non-blocking Tastatur-Lesen mit GETIN

; Nicht-blockierendes Tastatur-Lesen für Game-Loop
ReadKey:
       JSR $FFE4          ; GETIN: A = Taste (0 = keine)
       CMP #$00
       BEQ .no_key        ; kein Tastendruck
       CMP #$1D           ; CRSR-RIGHT?
       BEQ .move_right
       CMP #$9D           ; CRSR-LEFT?
       BEQ .move_left
.no_key:
       RTS

Datei-I/O — Laden, Speichern, Kanäle

Datei-I/O folgt immer der Reihenfolge: SETLFS → SETNAM → OPEN (optional) → CHKIN/CHKOUT → Daten lesen/schreiben → CLOSE. Für LOAD/SAVE ist die Sequenz kürzer: SETLFS → SETNAM → LOAD/SAVE. Nach jedem I/O-Aufruf STATUS ($90) prüfen.

AdresseNameEingangAusgangBeschreibung
$FFBASETLFS A=LFN (1–127), X=Gerät (8=Disk, 4=Drucker), Y=SA (0=kein, 1=ohne Name, 2=PRG, 15=Befehlskanal) — Logische Dateinummer, Gerätenummer und Sekundäradresse setzen. Muss vor OPEN/LOAD/SAVE aufgerufen werden
$FFBDSETNAM A=Länge, X=Lo-Adresse Name, Y=Hi-Adresse Name — Dateinamen setzen. Bei LOAD ohne Namen: A=0. Maximale Länge: 16 Zeichen
$FFC0OPEN — C=1: Fehler (A=Fehlercode) Datei öffnen (nach SETLFS + SETNAM). STATUS ($90) prüfen
$FFC3CLOSE A=LFN (logische Dateinummer) — Datei schließen
$FFC6CHKIN X=LFN C=1: Fehler Geöffnete Datei als Eingabekanal setzen. Danach liest CHRIN/GETIN von dieser Datei
$FFC9CHKOUT X=LFN C=1: Fehler Geöffnete Datei als Ausgabekanal setzen. Danach schreibt CHROUT in diese Datei
$FFCCCLRCHN — — Alle Kanäle zurücksetzen. Eingang → Tastatur, Ausgang → Bildschirm
$FFD5LOAD A=0 laden / A=1 verify, X=Lo-Zieladresse, Y=Hi-Zieladresse X/Y = Endadresse+1, C=1: Fehler Datei laden. Sekundäradresse=0: Ladeadresse aus Datei-Header. SA=1: lade an X/Y-Adresse
$FFD8SAVE A=ZP-Zeiger auf 2-Byte-Startadresse, X=Lo-Endadresse+1, Y=Hi-Endadresse+1 C=1: Fehler Speicherbereich auf Gerät sichern
$FFE7CLALL — — Alle offenen Dateien schließen (interne Tabellen zurücksetzen)
$FFB7READST — A=STATUS ($90) I/O-Statuswort lesen. Bit 6=EOF/EOI, Bit 7=Gerät nicht vorhanden
STATUS ($90) — Bit-Bedeutung Bit 0 = 1: Timeout beim Schreiben
Bit 1 = 1: Timeout beim Lesen
Bit 6 = 1: EOF / End of File (EOI auf seriellem Bus)
Bit 7 = 1: Gerät nicht vorhanden (Device Not Present)

Datei laden — vollständige SETLFS → SETNAM → LOAD Sequenz

; Datei "LEVEL1" von Laufwerk 8 an feste Adresse $3000 laden
; Gibt C=1 zurück wenn Fehler

fname:  !pet "LEVEL1"
fnamelen = 6

LoadLevel:
        LDA #1             ; Logische Dateinummer 1
        LDX #8             ; Gerät 8 (Disk)
        LDY #1             ; Sekundäradresse 1 = lade an X/Y-Adresse
        JSR $FFBA          ; SETLFS

        LDA #fnamelen      ; Länge des Dateinamens
        LDX #<fname        ; Lo-Adresse des Namens
        LDY #>fname        ; Hi-Adresse des Namens
        JSR $FFBD          ; SETNAM

        LDA #0             ; A=0: laden (nicht verify)
        LDX #<$3000        ; Zieladresse Lo
        LDY #>$3000        ; Zieladresse Hi
        JSR $FFD5          ; LOAD
        BCS .error         ; C=1: KERNAL-Fehler

        JSR $FFB7          ; READST: Status lesen
        BNE .io_error      ; Status ≠ 0: I/O-Problem
        CLC
        RTS

.error:
.io_error:
        SEC                ; C=1 signalisiert Fehler
        RTS

Serieller Bus (IEC)

Der IEC-Bus verbindet den C64 mit Diskettenlaufwerken (Gerät 8/9) und Druckern (Gerät 4/5). Die höherstufigen Routinen (OPEN/LOAD/SAVE) nutzen diese Routinen intern. Direkter Zugriff ist selten nötig — z. B. beim Schreiben eines eigenen Fast-Loaders oder bei direkter Kommunikation mit dem Befehlskanal des Laufwerks.

AdresseNameEingangAusgangBeschreibung
$FFB1LISTEN A = Gerätenummer (8–30) — Gerät in LISTEN-Modus versetzen. Danach SECOND und Daten mit CIOUT senden
$FFB4TALK A = Gerätenummer — Gerät in TALK-Modus versetzen. Danach TKSA und Daten mit ACPTR empfangen
$FF93SECOND A = Sekundäradresse (nach LISTEN, ORed mit $60) — Sekundäradresse nach LISTEN senden
$FF96TKSA A = Sekundäradresse (nach TALK, ORed mit $60) — Sekundäradresse nach TALK senden
$FFA5ACPTR — A = empfangenes Byte Ein Byte vom seriellen Bus empfangen
$FFA8CIOUT A = Byte — Ein Byte auf den seriellen Bus senden (nach LISTEN/SECOND)
$FFABUNTLK — — UNTALK senden — beendet TALK-Sequenz
$FFAEUNLSN — — UNLISTEN senden — beendet LISTEN-Sequenz

Disk-Fehlerkanal öffnen und auslesen

; Disk-Fehlerkanal öffnen (Kanal 15 = Befehlskanal)
; Laufwerk 8, sekundäre Adresse 15

OpenCmdChannel:
        LDA #15            ; LFN 15
        LDX #8             ; Gerät 8
        LDY #15            ; SA 15 = Befehlskanal
        JSR $FFBA          ; SETLFS
        LDA #0             ; Kein Dateiname
        JSR $FFBD          ; SETNAM (leer)
        JSR $FFC0          ; OPEN
        RTS

; Fehlermeldung vom Laufwerk lesen
ReadDiskError:
        LDX #15            ; Befehlskanal
        JSR $FFC6          ; CHKIN
.loop:  JSR $FFCF          ; CHRIN: Byte lesen
        CMP #13            ; Return = Ende
        BEQ .done
        JSR $FFD2          ; CHROUT: auf Bildschirm ausgeben
        BNE .loop
.done:  JSR $FFCC          ; CLRCHN
        RTS

IRQ-Vektor KERNAL-konform setzen

; IRQ-Vektor KERNAL-konform über $0314/$0315 setzen
; Eigene Routine: MyIRQ

InstallIRQ:
        SEI                   ; Interrupts sperren während Umbiegung
        LDA #<MyIRQ
        STA $0314             ; IRQ-Vektor Lo
        LDA #>MyIRQ
        STA $0315             ; IRQ-Vektor Hi
        CLI                   ; Interrupts wieder freigeben
        RTS

MyIRQ:
        ; Eigene IRQ-Behandlung
        ; ...
        JMP $EA31             ; KERNAL-IRQ-Ende (Timing, Tastatur-Scan)
        ; Alternativ: JMP $EA7E (nur RTI, kein KERNAL)
$EA31 vs. $EA7E — Welcher Exit? JMP $EA31 ruft den vollständigen KERNAL-IRQ auf: Tastatur-Scan, TOD-Timer, Kassetten-Timing. Der IRQ dauert dadurch länger, aber alle KERNAL-Dienste bleiben aktiv.

JMP $EA7E macht nur den Raster-Acknowledge und RTI — minimaler Overhead, kein Tastatur-Scan, kein TOD-Update.

Faustregel: Demo-IRQs ohne Tastatur → $EA7E. Spiele mit Tastatur-Eingabe → $EA31.

Zeit & Jiffy-Timer

Der KERNAL-Jiffy-Timer zählt bei PAL mit 50 Hz — jeder IRQ inkrementiert den 3-Byte-Zähler um 1. Nach 24 Stunden (86.400 Frames bei 50 Hz) läuft er auf 0 zurück. Der Timer eignet sich für einfaches Timing, Highscore-Zeitlimits oder Demo-Sync. Für präziseres Timing die CIA-Timer verwenden.

AdresseNameEingangAusgangBeschreibung
$FFDBSETTIM A=High, X=Mid, Y=Low — Jiffy-Timer setzen. 3-Byte-Wert, High-Byte nach A
$FFDERDTIM — A=High, X=Mid, Y=Low Jiffy-Timer lesen. PAL: ~50 Ticks/Sek. NTSC: ~60 Ticks/Sek
$FFEAUDTIM — — Jiffy-Timer um 1 inkrementieren. Wird normalerweise vom KERNAL-IRQ aufgerufen
$FFE1STOP — C=1: RUN/STOP gedrückt RUN/STOP-Taste prüfen. Nützlich für Notausstieg in Endlosschleifen

Timer lesen — Sekunden seit Spielstart (PAL)

; Einfaches Timing: Sekunden seit Spielstart (PAL)
; Am Spielstart: JSR ResetTimer
; Zum Lesen: JSR GetSeconds → A = Sekunden (0–255)

ResetTimer:
        LDA #0
        LDX #0
        LDY #0
        JSR $FFDB    ; SETTIM: Timer auf 0 zurücksetzen
        RTS

GetSeconds:          ; Gibt Sekunden seit ResetTimer zurück (PAL: /50)
        JSR $FFDE    ; RDTIM: A=Hi, X=Mid, Y=Lo
        ; Für einfaches Sekunden-Zählen: nur Mid-Byte / 50
        ; (vereinfacht, ~5% Fehler bei 50 Hz)
        TXA          ; Mid-Byte nach A
        RTS          ; A ≈ Sekunden (bei kurzen Zeiten)

★ Skill-Benchmark: Anhang H

Du hast die KERNAL-Routinen verinnerlicht, wenn du folgendes kannst:

  • Du schreibst die SETLFS → SETNAM → LOAD Sequenz für eine Datei von Gerät 8 ohne Nachschlagen korrekt auf
  • Du baust GETIN in einen Game-Loop ein und reagierst auf Cursor-Tasten non-blocking
  • Du prüfst STATUS ($90) nach LOAD korrekt auf EOF (Bit 6) und Gerät-nicht-vorhanden (Bit 7)
  • Du hängst deinen eigenen IRQ über $0314/$0315 ein (mit SEI/CLI) und entscheidest korrekt zwischen $EA31 und $EA7E als Exit
  • Du weißt welchen Wert $01 haben muss damit KERNAL erreichbar ist ($37 = Standard) und wie du prüfst ob es aktiv ist
  • Du liest den Disk-Fehlerkanal (LFN 15, SA 15) mit OPEN/CHKIN/CHRIN/CLRCHN korrekt aus
