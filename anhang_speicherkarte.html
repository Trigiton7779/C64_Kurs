C64 Speicherkarte

$0000 – $FFFF 64 KB Adressraum Klick auf Region für Details

Der C64 hat einen 16-Bit-Adressraum von $0000 bis $FFFF (65536 Bytes). Durch Bank Switching kann RAM hinter ROM-Bereichen liegen — gesteuert über Register $0001.

Bank-Switching — Register $0001

Das I/O-Port-Register $0001 (6510-interner Port, Richtungsregister in $0000) steuert, welche ROMs/RAM aktiv sind:

$0001 WertBit 2 (KERNAL)Bit 1 (BASIC)Bit 0 (I/O/Charset)Konfiguration
$37 = %00110111ROMROMI/OStandard-Boot-Zustand (BASIC+KERNAL+I/O)
$36 = %00110110ROMROMCharset-ROMZeichensatz-ROM bei $D000 sichtbar
$35 = %00110101ROMRAMI/OBASIC aus, KERNAL+I/O an — typisch für Spiele
$34 = %00110100ROMRAMCharset-ROMBASIC aus, Zeichensatz sichtbar
$33 = %00110011RAMROMI/OSelten: KERNAL aus, BASIC an
$30 = %00110000RAMRAMRAMAlles RAM — $D000 = normales RAM!
$31 = %00110001RAMRAMI/OKERNAL+BASIC aus, I/O an — für Demos
WICHTIG: Bits 3–5 von $0001
Bits 3–5 steuern den Kassetten-Port und sollten nicht verändert werden: immer %00110xxx verwenden. Der Standard-Wert für Spielprogramme: LDA #$35 : STA $01.

VIC-II Speicher-Banken

Der VIC-II kann nur innerhalb seiner 16-KB-Bank sehen. Die Bank wird über CIA2-Register $DD00 Bits 0–1 gewählt (invertiert!). Standard ist Bank 0 ($0000–$3FFF).

$DD00 Bits 1-0VIC-BankAdressbereichZeichensatz-Standard
%11 (3)Bank 0$0000–$3FFFCharset bei $1000 (Standard!)
%10 (2)Bank 1$4000–$7FFFKein ROM-Charset
%01 (1)Bank 2$8000–$BFFFCharset bei $9000
%00 (0)Bank 3$C000–$FFFFKein ROM-Charset (I/O überlagert)
 
; VIC-Bank 1 aktivieren ($4000-$7FFF)
          LDA $DD00
          AND #$FC         ; Bits 0-1 löschen
          ORA #$02         ; %10 = Bank 1
          STA $DD00

; Screen-RAM bei $4000, Zeichensatz bei $4800 ($D018 anpassen):
          LDA #%00010010   ; Screen=$4000 (Bits 7-4 = %0001), Charset=$4800 (Bits 3-1 = %001)
          STA $D018
